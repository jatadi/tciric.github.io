Cloud AutoTagger

A modern serverless application showcasing event-driven architecture and cloud-native patterns

Project Overview

Cloud AutoTagger demonstrates modern cloud architecture patterns through a practical application that automatically extracts and stores metadata from files uploaded to S3. Built entirely with serverless technologies, it showcases event-driven design, infrastructure as code, and DevOps best practices.

Above: DynamoDB entries showing automatic metadata extraction from both JSON and CSV files. Note how the system adapts to different file formats while maintaining consistent metadata structure.

System Architecture

Core Components

  • S3 Bucket - File storage with versioning
  • Lambda Function - Serverless processing
  • DynamoDB - NoSQL metadata storage
  • Event Triggers - Automatic processing

Infrastructure

  • Terraform - Infrastructure as Code
  • GitHub Actions - CI/CD pipeline
  • AWS IAM - Security policies
  • CloudWatch - Monitoring & logs

Above: CloudWatch logs demonstrating the complete event flow - from file upload detection to successful metadata storage. Note the sub-second processing times for both file types.

Key Features

Technical Features

  • Format Flexibility - JSON and CSV support
  • Real-time Processing - Event-driven architecture
  • Automated Deployment - CI/CD pipeline
  • Cost Efficiency - Pay-per-use model

Cloud Practices

  • Serverless Design - No server management
  • Event-Driven - Loose service coupling
  • Infrastructure as Code - Version controlled
  • Security First - Least privilege access

Implementation Details

The project implements a sophisticated yet maintainable architecture where file uploads trigger automatic metadata extraction. The Lambda function processes the files based on their format, extracts relevant metadata, and stores it in DynamoDB. The entire infrastructure is defined in Terraform, ensuring reproducible deployments and environment consistency.

Learning Outcomes

This project provided hands-on experience with modern cloud development practices, including serverless architecture, event-driven design, infrastructure as code, and CI/CD pipelines. It demonstrates practical implementation of cloud-native patterns and best practices in a real-world scenario.
